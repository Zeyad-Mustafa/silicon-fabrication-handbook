# Python Simulation Examples

This folder contains the Python-based computational models and 3D visualizations for the Silicon Fabrication Handbook. It includes device physics, process visualization, and MEMS system dynamics. The scripts are written to run standalone and generate plots or figures, with outputs stored under `simulation-examples/python/images/`.

## Quick Start

1. Create and activate a Python environment.
2. Install dependencies:

```bash
pip install -r requirements.txt
```

3. Run any script directly, for example:

```bash
python oxide_growth_model_3D.py
python mosfet_iv_curves_3D.py
python resonator_response_3D.py
```

## What Each File Does

| File | Purpose | Primary Outputs |
| --- | --- | --- |
| `oxide_growth_model_3D.py` | Deal-Grove thermal oxidation model with temperature, ambient, and orientation sweeps plus 3D growth visualization. | Oxide growth plots and 3D cross-sections in `images/oxide/`. |
| `mosfet_iv_curves_3D.py` | MOSFET I-V modeling across linear/saturation/subthreshold regions, with 3D device rendering and parameter extraction. | I-V curves, small-signal parameters, and 3D plots in `images/mosfet/`. |
| `resonator_response_3D.py` | MEMS resonator frequency response, damping models, Duffing nonlinearities, and sensitivity sweeps. | Bode/Nyquist plots, damping comparison, 3D structure, and sweep results in `images/resonator/`. |
| `comb_drive_analysis_3D.py` | Electrostatic comb-drive actuator analysis with force, capacitance, and displacement calculations. | Plots rendered at runtime (no fixed output directory). |
| `mems_spring_mass.py` | Spring-mass-damper MEMS accelerometer with time response, Bode plot, and noise analysis. | Plots saved under `images/` (relative to the run directory). |
| `mems_spring_mass.ipynb` | Notebook version of the spring-mass MEMS simulation. | Interactive exploration in Jupyter. |
| `capacitive_sensor_3d.py` | 3D visualization of a capacitive accelerometer with proof mass, electrodes, and springs. | 3D figure saved under `images/` (relative to the run directory). |
| `chip_fabrication_3d.py` | Layer-by-layer CMOS/MOSFET fabrication process visualization. | Step-by-step 3D figures saved under `images/` (relative to the run directory). |
| `thermal_actuaror_sim3d2.py` | Python-based 3D thermal actuator with live animation and UI widgets. | Matplotlib animation window (no fixed output directory). |
| `thermal_actuator_sim3d.py` | React + Three.js 3D thermal actuator demo (not a Python script). | Runs in a JS/React environment. |
| `mems-simulation/` | Additional MEMS-related resources and supporting material. | Files in that subfolder. |

## Notes

- Scripts that write to `images/` without a subfolder (`capacitive_sensor_3d.py`, `mems_spring_mass.py`, `chip_fabrication_3d.py`) save relative to the directory you run them from.
- For the React/Three.js demo in `thermal_actuator_sim3d.py`, you will need a JS toolchain to run it. It is not executed by Python.
